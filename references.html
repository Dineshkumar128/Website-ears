Academic references — Appendix K

Sources that informed EARS

Standards and research that underpinned our design decisions — from PWA specs to accessibility guidelines and security practices.

01
Progressive Web Apps — Web.dev
Google Developers (2024). Progressive Web Apps. web.dev/progressive-web-apps. Foundational guidance for Service Worker implementation, Web App Manifests and offline-first architecture — used directly in sw.js and manifest.json.
PWA · SERVICE WORKER
02
Service Worker API — MDN Web Docs
Mozilla Developer Network (2024). Service Worker API. developer.mozilla.org. Used to understand install, activate, and fetch event lifecycle — directly reflected in sw.js event handler design.
SERVICE WORKER · CACHE API
03
Web Application Manifest — W3C Specification
W3C (2023). Web Application Manifest. w3.org/TR/appmanifest. Guided manifest.json — name, short_name, display: standalone, icons, and theme_color fields.
PWA · MANIFEST
04
OWASP — XSS Prevention Cheat Sheet
OWASP Foundation (2024). Cross Site Scripting Prevention Cheat Sheet. cheatsheetseries.owasp.org. Informed the esc() sanitisation function — encoding &, <, >, and " in all dynamically rendered user content.
SECURITY · XSS
05
Web Content Accessibility Guidelines (WCAG) 2.1
W3C (2018). WCAG 2.1. w3.org/TR/WCAG21. Guided semantic HTML, contrast ratios, and ARIA considerations. The 72/100 mobile score represents a target for improvement in line with these guidelines.
ACCESSIBILITY · WCAG
06
localStorage API — MDN Web Docs
Mozilla Developer Network (2024). Window.localStorage. developer.mozilla.org. Informed the DB abstraction layer — JSON serialisation patterns, storage limits, and the namespacing strategy.
DATA LAYER · STORAGE
07
Google Lighthouse — Performance Auditing
Google Chrome Developers (2024). Lighthouse. developer.chrome.com/docs/lighthouse. Primary automated quality tool. EARS achieved 100/100 on Performance, Best Practices and SEO on desktop.
QUALITY · PERFORMANCE
08
Google Apps Script — Web Apps Documentation
Google Developers (2024). Google Apps Script: Web Apps. developers.google.com/apps-script/guides/web. Informed the Sheets sync architecture — doPost() endpoint design, CORS handling, and JSON response format expected by syncToGoogleSheets().
GOOGLE SHEETS · SYNC
09
Fetch API — MDN Web Docs
Mozilla Developer Network (2024). Fetch API. developer.mozilla.org. Used for the Google Sheets sync — async/await fetch with POST, JSON headers, and error handling that preserves UX on failure.
FETCH · ASYNC
10
Blob API and File Download — MDN Web Docs
Mozilla Developer Network (2024). Blob, URL.createObjectURL(). developer.mozilla.org. The technique behind exportDataAsJSON() — creating a downloadable file from in-memory data without a server.
DATA EXPORT · BLOB
