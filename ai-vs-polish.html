Our Polish
5-rule enforced validation
We implemented validatePassword() with five independent regex rules — length, uppercase, lowercase, digit, and special character. Each failure returns a specific, actionable error message.
// Our version — 5 independent checks
if (pw.length < 8) return 'Min 8 chars';
if (!/[A-Z]/.test(pw)) return 'Needs uppercase';
if (!/[0-9]/.test(pw)) return 'Needs a number';
if (!/[^A-Za-z0-9]/.test(pw)) return 'Needs special char';
return null; // valid

AI Generated
innerHTML without sanitisation
User-provided data was injected directly into innerHTML in multiple places throughout the dashboard and activity list. Any user could have injected arbitrary HTML or JavaScript into the page.
// AI version — XSS vulnerability
card.innerHTML = `<h3>${item.title}</h3>`;
// ↑ if title = "<script>alert(1)</script>"
// → executes in browser
XSS vulnerability present
Our Polish
esc() wraps every output
We created the esc() helper that HTML-encodes &, <, >, and " before any dynamic value touches the DOM. Every single template literal uses it. Verified passing by external security reviewer.
// Our version — sanitised
function esc(v) {
  return String(v || '')
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;');
}
card.innerHTML = `<h3>${esc(item.title)}</h3>`;

Our Polish
Full PWA certification
We wrote sw.js and manifest.json from scratch. The Service Worker pre-caches all assets on install and responds to all fetch events — cache-first, network fallback. Lighthouse PWA: certified.
// sw.js — written from scratch
const CACHE_NAME = 'ears-v2';
self.addEventListener('install', e => {
  e.waitUntil(
    caches.open(CACHE_NAME)
      .then(c => c.addAll(urlsToCache))
  );
});

AI Generated
Scattered localStorage calls
localStorage.getItem() and setItem() were scattered across 20+ functions. No consistent key naming, no JSON parse/stringify wrapper, and no null safety — crashes when a key didn't exist yet.
// AI — localStorage scattered everywhere

const acts = JSON.parse(data);
// ↑ crashes if 'activities' is null
No abstraction layer
Our Polish
DB object — clean data API
We designed a DB object that wraps all localStorage access, handles JSON parse/stringify, namespaces all keys with 'ears3_', provides typed accessors with safe defaults, and includes an auto-incrementing ID system.
// Our DB — single abstraction layer
const DB = {
  get: k => JSON.parse(
    localStorage.getItem('ears3_'+k) || 'null'
  ),
  acts: () => DB.get('acts') || [], // safe
  nextId: key => { ... } // auto-inc
};

AI Generated
No audit trail
No logging of any user actions. No record of who logged in, who created which activity, or when anything changed. An admin had no way to review system history or investigate discrepancies.
// AI version
function saveActivity(data) {
  const acts = getActivities();

No audit trail
Our Polish
Full audit log system
We added addAudit() called on every significant action — login, logout, register, activity create/update/delete, admin actions. The log is capped at 200 entries and visible to admins in the admin panel.
// Our version — every action logged
function addAudit(action, details) {
  const logs = DB.audit();
  logs.unshift({ id, at: nowIso(),
    actor: currentUser().email,
    action, details });
  DB.saveAudit(logs.slice(0, 200)); // capped
}

Development journey

When we made each change

Our polishing work happened across four sprints — each sprint tackling a different layer of quality.

Sprint 1 — Foundation
AI scaffold → working CRUD
Started with the AI-generated base. Established the DB abstraction layer, fixed localStorage key naming, added safe defaults throughout, and implemented the auto-incrementing ID system. This sprint was about making the data layer trustworthy.
Architecture
Sprint 2 — Security
Password policy + XSS protection
Added validatePassword() with five rules, validateEmail() with RFC compliance, and esc() for XSS sanitisation across all DOM outputs. Added role-based route guards (isAdmin()) and session management. External reviewer later confirmed XSS protection was effective.
Security
Sprint 3 — PWA + Performance
Service Worker + manifest + cache strategy
Wrote sw.js and manifest.json from scratch. Defined the cache-first fetch strategy, implemented the versioned cache key system, and integrated Service Worker registration into boot(). Achieved 100/100 Lighthouse performance score and PWA certification.

Google Sheets pipeline + JSON backup
Engineered the full Google Sheets sync pipeline (with guard clauses, incremental sync, and merge logic), JSON export with password redaction, and intelligent import with conflict resolution. Added the audit log system. Achieved 0 W3C validation errors.
