Feature Breakdown

What we built,
and why.

Five deliberate engineering choices that take EARS from a simple form to a production-ready system — with a clear path to grow.

01 — Progressive Web App

Install it.
Use it offline.

EARS is a certified PWA — installable on any Android, iOS or desktop device. After first visit, it runs completely without internet.

  • manifest.json defines app name, icons, theme colour and standalone display mode
  • sw.js registers a Service Worker that intercepts every network request
  • On install, all critical assets are pre-cached with a single cache.addAll() call
  • Subsequent requests answered from cache first — zero network latency on repeat visits
  • Chrome, Edge and Safari all offer "Add to Home Screen" automatically

02 — Security & Authentication

Login that
doesn't roll over.

Authentication with a strong password policy enforced at registration. All rendered content is sanitised to block XSS injection — verified by external reviewer.

  • validatePassword() tests 5 independent rules before accepting any password
  • validateEmail() enforces RFC-compliant format before any lookup
  • esc() HTML-encodes every dynamic string before DOM injection
  • Sessions stored in localStorage with userId and loggedInAt timestamp
  • Role-based routing — isAdmin() guards every admin page access

03 — Offline Cache Storage

Works without
Wi-Fi.

The Cache API holds a complete snapshot of the app. After first load, EARS runs entirely from local cache. Data changes persist in localStorage — nothing lost offline.

  • install event pre-caches the full asset list before worker activates
  • fetch event: cache hit returns instantly, miss falls back to network
  • Versioned cache key ("ears-v2") — clean updates, old cache replaced atomically
  • navigator.onLine checked before sync — no failed calls when offline

04 — Google Sheets Sync

One URL away
from live sync.

The full pipeline is written, tested and dormant. Configure a Google Apps Script URL in admin settings — two-way sync activates immediately, no code changes needed.

  • syncToGoogleSheets() posts filtered JSON payload to Apps Script endpoint
  • downloadFromGoogleSheets() pulls remote data and merges by ID
  • SYNC_CONFIG controls excluded types, statuses, interval and retry count
  • lastSyncTime enables incremental sync — only changed records sent
  • Three guard clauses: no URL, concurrent sync, or offline all exit cleanly

05 — JSON Export & Import

Your data,
always portable.

Because localStorage is device-scoped, EARS includes a complete backup system. Export produces a clean JSON file. Import merges with conflict resolution.

  • exportDataAsJSON() serialises all activities + users (passwords redacted)
  • Blob API creates a downloadable file — entirely in-browser, no server
  • Import reads via FileReader, parses JSON, merges by ID
  • Conflict resolution: duplicate IDs prompt overwrite-or-skip
  • Filename: EARS_backup_YYYY-MM-DD.json
